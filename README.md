# PFC手帖 — セットアップ手順

オフラインで動くPWA（Webアプリ）です。Android端末にホーム画面アイコンとして
インストールでき、ネイティブアプリのように起動できます。

## 1. ファイルをホスティングする

スマホのブラウザから `file://` で直接開くと、保存機能（IndexedDB）や
オフラインキャッシュ（Service Worker）が正しく動かない場合があります。
以下のいずれかの方法で、簡単なWebサーバー経由で配信してください。

### おすすめ: GitHub Pages（無料・恒久的に使える）
1. GitHubに新しいリポジトリを作成
2. `pfc-app` フォルダの中身（index.html, manifest.json, sw.js, icon.svg,
   icon-192.png, icon-512.png）をすべてアップロード
3. リポジトリの Settings → Pages で公開を有効化
4. 発行されたURL（例: `https://あなたのID.github.io/リポジトリ名/`）に
   スマホのChromeでアクセス

### 簡易テスト用: ローカルネットワークで試す
PC側で以下を実行し、同じWi-FiにいるスマホからPCのIPアドレスでアクセスします。
```
cd pfc-app
python3 -m http.server 8000
```
スマホのブラウザで `http://(PCのIPアドレス):8000/index.html` を開く

## 2. ホーム画面に追加する（Android / Chrome）

1. アプリのURLをChromeで開く
2. 右上の「⋮」メニューをタップ
3. 「ホーム画面に追加」または「アプリをインストール」を選択
4. 以後はホーム画面のアイコンから、ブラウザのUIなしで起動できます

## 3. 使い方

- **今日タブ**: 右下の「＋」から食事を記録。朝食/昼食/夕食/間食を選び、
  マイ食品から選択するだけです。円グラフで今日のPFCバランスが一目でわかります。
- **設定タブ**: 体重・体脂肪率を記録すると、基礎代謝（Katch-McArdle式）が自動計算されます。
  1日の目標カロリーは「基礎代謝のみ」「活動量を加味（基礎代謝×活動係数）」のどちらかを
  選んで使えます。PFCの配分（g数）は体重と目標カロリーから自動算出され、
  好みに応じてスライダーで微調整も可能です（「自動算出値に戻す」でいつでも復帰できます）。
- **履歴タブ**: 過去の記録を日付ごとに一覧表示。タップするとその日の詳細に移動します。
- **ブックマークタブ**: 食品選択画面の★をタップして登録した、よく使う食品の一覧です。
- **マイ食品の登録**: 食事記録シートの「＋ 新しい食品をマイ食品に登録する」からは、
  登録と同時にその場で今日の記録にも追加できます（時間帯を選んで「登録して今日の記録に追加」）。
  設定タブの「マイ食品を編集する」からは、記録には追加せず登録・編集・削除のみ行えます。
  P・F・Cグラム数を入れるとカロリーは自動計算されます。
- **推移グラフ**: 設定タブ内に、体重・体脂肪率・基礎代謝の推移グラフがあります。
  期間は1ヶ月／3ヶ月／すべてから切り替えられます。

## 4. データについて

- すべてのデータは**端末のブラウザ内（IndexedDB）にのみ**保存されます。
  他の端末やクラウドとは同期されません。
- 設定タブの「バックアップを書き出す」でJSONファイルとして保存し、
  「バックアップから復元する」で読み込めます。機種変更時やブラウザの
  データ消去前に、バックアップを取ることをおすすめします。
- ブラウザの「サイトデータを削除」やキャッシュ消去を行うと、
  保存した記録が失われる可能性があるのでご注意ください。

## 5. 今後の拡張について

「AIカロリー推定」（写真判定・会話形式での推定）は今回見送りましたが、
食品データの構造（名前・カテゴリ・PFC・カロリー）はそのまま流用できる設計に
してあります。将来的にAI機能を追加する際は、推定結果を「マイ食品登録」フォームに
自動入力する形で連携しやすくなっています。
